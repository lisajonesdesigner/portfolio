Components of a User Flow

Components of a User Flow

Lesson 2 of 4

Key Components

User flows break tasks into smaller elements. These elements help describe movement, decisions, and interactions.

Common Components

• Start point
• End point
• Steps or actions
• Decision points
• Inputs or outputs
• Arrows that show direction

How components work together

A user begins at a defined entry point, completes one step, moves to another, and reaches a decision moment if needed. Arrows show the correct direction. The flow ends when the user completes the task or exits the path.

Primary and Alternate Paths

A user flow includes both the main path and any alternate paths. These alternate paths show optional screens, errors, or extra conditions.
