UI Design Patterns

UI Design Patterns

Lesson 1 of 4

Don't Reinvent the Wheel

UI Patterns are standard solutions to common problems. Users expect them to work a certain way. By using established patterns, you reduce the user's "cognitive load" (brain power required to figure things out).

I

Common Patterns

Input: Dropdowns, checkboxes, toggles.

Navigation: Tabs, breadcrumbs, burger menus.

Feedback: Progress bars, success toasts, loaders.

     

Breadcrumbs and buttons in a design system

Benefits of UI Patterns

  • Reduce cognitive effort
  • Improve prediction
  • Speed up design
  • Support accessibility
  • Strengthen consistency

I

Pattern Types

  • Navigation patterns. Tabs, menus, search.
  • Information patterns. Cards, progress bars, notifications.
  • Input patterns. Forms, dropdowns, radio buttons.

I

Choosing Patterns

Base the choice on user goals and task requirements. Choose patterns that simplify the experience.
