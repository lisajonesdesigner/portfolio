Assessment 5

Prototype and Testing

Lesson 4 of 4

Overview

You build a low-fidelity clickable prototype directly in Miro. You design your onboarding screens using Miro shapes, link them together, and complete a short user test with a partner.

I

Part 1. Create your screens

  1. Open the shared Miro board.

  2. Add a new frame titled “Prototype Screens”.

  3. Create three to six mobile-screen frames using the Phone Frame shape.

  4. Use rectangles, circles, lines, and text blocks to build each screen.

  5. Follow your NSURE user flow and place one screen per step.

  6. Keep layouts simple and focused on structure.

I

Part 2. Make the prototype clickable

  1. Select a screen.

  2. Add hotspots using the “Link to” tool.

  3. Place hotspots over buttons, links, or next-step actions.

  4. Link each hotspot to the correct target screen.

  5. Test each link to confirm movement.

  6. Keep interactions clear and minimal.

I

Part 3. Run a user test

  1. Share the Miro board link with a partner.

  2. Ask them to complete a short onboarding task.

  3. Observe without guiding them.

  4. Note hesitation, confusion, or unexpected actions.

  5. Switch roles and test again.

I

Part 4. Save and upload

  1. Test all hotspots.

  2. Export the prototype frame or share the Miro link.

  3. Save it for assessment upload.

I

This activity forms one part of the Session 3 assessment.
